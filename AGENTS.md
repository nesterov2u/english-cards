# Правила работы с English Cards

## Принципы

- Рабочая директория: `/Users/andrew/Documents/English-Cards`.
- Сначала вносить изменения локально и проверять затронутый сценарий на `http://127.0.0.1:4173/`.
- Интерфейс остаётся на русском, минималистичным и единообразным; не возвращать удаленный раздел «Настройки» или кнопку добавления в словарь без запроса.
- Не добавлять secret key, service role key, токены или другие секреты. Для статического браузерного приложения допустим только publishable/anon key при настроенном RLS.
- Не менять Supabase-схему, RLS или текущую открытую модель доступа без отдельного запроса.

## Структура

- `index.html` — разметка, три раздела и диалог редактирования.
- `styles.css` — дизайн-токены, карточки, поля и адаптивность.
- `app.js` — Supabase REST API, автозаполнение через Wiktionary, подгрузка дополнительных значений, форм неправильных глаголов, учебная сессия и клавиатурное управление.
- `wiktionary.js` — чистые функции разбора Wiktionary; `tests/wiktionary_test.js` — локальные фикстуры и тесты парсера.
- `manifest.webmanifest`, `icon-*.png`, `logo-outline.svg` — PWA и бренд.
- `supabase/schema.sql` — схема таблицы и RLS-политики.
- `.github/workflows/deploy-pages.yml` — публикация GitHub Pages.

## UX, который нужно сохранять

- Порядок разделов: «Перевод», «Словарь», «Учить»; на мобильных все три пункта всегда помещаются в одну строку нижней навигации.
- «Перевод» — отдельная форма добавления: английское слово и перевод редактируемые; другие значения, формы неправильных глаголов и синонимы выводятся только текстом в этом порядке. Если пользователь вручную меняет перевод, все три автоданных очищаются. В словаре нет верхней кнопки `+`.
- В «Словаре» дополнительные блоки идут в порядке «Другие значения» → «Формы» → «Синонимы» и используют одинаковый размер шрифта; другие значения показываются акцентным цветом без подписи, а формы и синонимы — серым, при этом у синонимов есть подпись «Синонимы».
- Диалог редактирования — как карточка и содержит только метку «Редактирование карточки», два поля (слово и перевод), CSS-крестик и контурную кнопку «Сохранить изменения». При ручной правке перевода скрытые автоданные, включая формы, очищаются.
- Учебная карточка: первый тап/`Enter`/`Space` — переворот, следующий — новая карточка. На обратной стороне показываются только основной перевод, другие значения и синонимы; примеры там не показываются. Сессия перемешанная и без повторов; в конце — финальная карточка и «Перемешать».
- Поддерживать loader при первоначальной загрузке и при открытии учебного режима.
- Для автозаполнения использовать английский Wiktionary как основной источник перевода, синонимов и форм. Русский Wiktionary допустим только как финальный резерв для перевода, если английская запись не содержит русского варианта. В приложении показывать только формы неправильных глаголов, если они есть; примеры предложений не выводить.
- Улучшать поиск только универсальными правилами разбора: секциями частей речи, первой этимологией, леммами, смысловыми блоками перевода, страницами `/translations`, MediaWiki search и очисткой разметки. Не добавлять исключения, хардкод переводов или списки слов для конкретных слов.
- Для обычных слов отдавать приоритет существительному, затем прилагательному, наречию и глаголу; для распознанных форм глагола сначала искать самостоятельное значение формы, затем лемму. Не смешивать части речи в резервном поиске перевода.

## Проверка

1. Открыть локальную страницу и проверить изменённый сценарий.
2. При изменениях данных убедиться, что индикатор показывает подключение и карточки загружаются.
3. При UI-изменениях проверить мобильную ширину и нижнюю навигацию.
4. Перед коммитом запустить `git diff --check`, `deno check app.js` и `deno test tests/wiktionary_test.js` при изменениях парсера.
5. Не включать в коммит посторонние изменения пользователя. Перед staging сверить `git status --short`.

## Git

- Коммит — только после явного подтверждения, что локальная версия готова.
- Если пользователь говорит «сделай коммит», создать **локальный** коммит.
- Push на GitHub выполнять только при явном упоминании GitHub/push/отправки.
