# Контекст проекта

## Продукт

**English Cards** — личное веб-приложение для изучения английских слов и фраз. Основной путь: добавить слово в разделе «Перевод», при необходимости поправить автоподставленные данные, затем повторять перемешанную колоду в разделе «Учить».

## Технологии

- Статические `index.html`, `styles.css`, `app.js`, без сборщика.
- Supabase REST API — общая база карточек.
- Английский Wiktionary MediaWiki API — основной источник автозаполнения: перевод, примеры, синонимы и формы неправильных глаголов. Русский Wiktionary применяется только как финальный резерв для перевода, когда в английской записи нет русского варианта.
- PWA: `manifest.webmanifest`, иконки и стартовый `theme-color: #00BDBD`; после загрузки приложения `app.js` меняет его на `#f5f5f7`.
- GitHub Pages — публикация после push в `main`.

## Интерфейс и визуальная система

- Язык интерфейса — русский; стиль — лёгкий, минималистичный, с ориентиром на Apple Translate.
- Акцентный цвет — `#00B8C8`, фон — `#f5f5f7`, поверхность карточек и полей — белая.
- Разделы в порядке: **«Перевод»**, **«Словарь»**, **«Учить»**. На мобильных это одна нижняя навигация из трёх равных пунктов.
- Логотип приложения и заставки — `brand-logo.svg`; в верхней панели он окрашивается акцентным цветом через CSS mask. Справа — индикатор подключения к базе: акцентный при подключении, красный при ошибке.
- В «Переводе» нет общей белой панели: форма размещена на сером фоне. Английское слово и перевод — отдельные белые редактируемые поля; примеры, другие значения и синонимы отображаются обычным текстом.
- В «Переводе» другие значения акцентные, синонимы серые. При первом ручном изменении перевода все автоданные (примеры, другие значения, синонимы) очищаются.
- В «Словаре» есть заголовок «Словарь», карточки, редактирование (карандаш), удаление (крестик) и кнопка «Показать ещё» только когда есть следующая страница.
- В карточках «Словаря» другие значения подгружаются из Wiktionary, отображаются без подписи акцентным цветом; синонимы показываются серым с подписью. Запросы ограничены тремя параллельными и кэшируются на сессию.
- Диалог редактирования оформлен как карточка и содержит только метку «Редактирование карточки», два редактируемых поля, крестик и кнопку «Сохранить изменения». Примеры и синонимы хранятся скрыто, чтобы сохраниться без ручной правки перевода; дополнительные значения в словаре вычисляются отдельно.

## iPhone PWA и safe area

- Установленное на экран «Домой» приложение — отдельный standalone-WebKit-контекст, его нельзя проверять только в обычном Safari или на десктопе.
- В `<meta name="viewport">` сохранять `viewport-fit=cover`, но `apple-mobile-web-app-status-bar-style` оставлять `default`. Режим `black-translucent` в сочетании с `viewport-fit=cover` имеет известную ошибку WebKit: у фиксированных элементов нарушается нижняя координата.
- Для заставки у `html`, `body` и `.app-loader` обязательно задавать явный `background-color: #06D3D3` вместе с вертикальным градиентом `#00BDBD → #06D3D3`. Нативная safe area использует именно цвет фона, а не изображение градиента, поэтому цвет фона должен совпадать с нижней точкой градиента.
- `html` и `body` должны иметь `min-height: 100vh` и `min-height: 100dvh`. Это обход ошибки standalone-WKWebView: при коротком содержимом, как в разделе «Перевод», `window.innerHeight` может ошибочно исключать safe area.
- Нижняя навигация на мобильном закреплена как `position: fixed` и использует только `bottom: max(8px, env(safe-area-inset-bottom))`. Не вычислять её позицию через `window.innerHeight` или `visualViewport` и не добавлять safe area дважды.
- При изменении PWA-метаданных, начального экрана или safe area проверять приложение именно после переустановки с экрана «Домой»: удалить старую иконку, дождаться деплоя Pages, открыть сайт в Safari и добавить заново.

## Учебный режим

- Карточки проходят одну перемешанную очередь без повторов в текущей сессии.
- Первый тап / `Enter` / `Space` переворачивает карточку; следующий — открывает следующую.
- Показывается прогресс «Показано N из M».
- После последней карточки появляется квадратная финальная карточка «Все слова изучены!» и кнопка «Перемешать» для новой сессии.
- Учебная карточка квадратная (`max-width: 610px`), длинные значения подстраиваются без изменения стандартного размера остальных слов.
- На обратной стороне учебной карточки не показываются примеры: выводятся основной перевод, другие значения без подписи и синонимы с отступом между блоками.

## Данные и ограничения

- Карточка содержит `id`, `word`, `translation`, `examples`, `synonyms`, `phonetic`, `created_at`. Поля `definition` и `example` больше не используются интерфейсом и не должны возвращаться в формы. `phonetic` временно хранит внутреннюю метку ручного перевода, когда нужно запретить повторную подгрузку автоданных.
- Начальная загрузка словаря — `PAGE_SIZE = 60`; кнопка «Показать ещё» подгружает следующую страницу.
- Автоматический поиск запускается после паузы при вводе английского слова; устаревшие запросы отменяются. Для фразовых глаголов используется тот же MediaWiki API.
- Парсер извлекает секции частей речи и выбирает подходящую структурно: обычные слова ранжируются как существительное → прилагательное → наречие → глагол; для глагольной формы сначала определяется лемма. Собственное значение формы, например `born`, имеет приоритет над переходом к лемме.
- Учитывается только первая этимология записи, чтобы редкие омонимичные формы из поздних этимологий не подменяли основное значение (например, `book` не должен интерпретироваться как форма `bake`).
- На странице `/translations` пропускаются служебные блоки наподобие «in questions» и «in negations»; выбирается первый смысловой блок. Поэтому `do` получает «делать», а не «ли».
- Если точная страница не дала перевода, выполняется MediaWiki search до пяти близких заголовков. Перевод собирается из выбранной части речи, а примеры и синонимы — из всей английской записи, чтобы не терять полезные материалы. Дополнительные русские значения извлекаются отдельно и очищаются от MediaWiki-разметки.
- Не добавлять исключения для отдельных слов. Новые ошибки решать только универсальными правилами: структурой Wiktionary, частями речи, этимологиями, леммами, ранжированием значений, поиском и очисткой разметки.
- Если перевод не найден после всех шагов, приложение сообщает об этом под полем слова; пользователь может заполнить поля вручную.
- Общая коллекция намеренно открыта без авторизации для личного приложения. Не менять RLS без отдельного запроса.
