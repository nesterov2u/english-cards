# English Cards

Минималистичное PWA для изучения английских слов и фраз. Карточки хранятся в общей базе Supabase: приложение подставляет перевод, примеры, другие значения и синонимы, а затем показывает слова в перемешанной учебной сессии.

## Возможности

- раздел «Перевод»: редактируемые слово и перевод, а примеры, другие значения и синонимы — в виде текста;
- автозаполнение через Wiktionary; ручное изменение перевода очищает все автоданные;
- раздел «Словарь» с постраничной подгрузкой, редактированием, удалением и дополнительными значениями;
- раздел «Учить» с перемешанной колодой без повторов, прогрессом и финальной карточкой; на обратной стороне выводятся перевод, другие значения и синонимы без примеров;
- управление учебной карточкой касанием, `Enter` или `Space`;
- адаптивный интерфейс, нижняя навигация на мобильных и PWA-иконки для iOS/macOS.

## Локальный запуск

Проект не требует сборки или зависимостей:

```bash
python3 -m http.server 4173
```

Затем откройте <http://127.0.0.1:4173>.

## iPhone PWA

Для установки: откройте опубликованный сайт в Safari → «Поделиться» → «На экран “Домой”». Проверять safe area, заставку и нижнюю навигацию нужно именно в этом standalone-режиме, а не во вкладке Safari.

- `viewport-fit=cover` включён для edge-to-edge интерфейса;
- заставка использует явный бирюзовый `background-color` и градиент, чтобы нижняя системная область не становилась белой;
- мобильная навигация закреплена у нижней safe area и не зависит от высоты содержимого раздела;
- не использовать `black-translucent` для `apple-mobile-web-app-status-bar-style` и не вычислять положение навигации через `window.innerHeight`: в standalone-WKWebView это приводит к неверному отступу на коротких экранах, в частности в разделе «Перевод».

После изменений в этих настройках удалите старую иконку с экрана «Домой» и добавьте сайт заново — iOS может сохранять PWA-метаданные и ресурсы.

## Данные

Подключение к общей базе уже задано в `app.js`. Для новой базы выполните [`supabase/schema.sql`](supabase/schema.sql) в Supabase SQL Editor и используйте только publishable/anon key.

- Английский Wiktionary — основной источник автозаполнения для слов и фраз: перевод, примеры, синонимы и формы неправильных глаголов. Русский Wiktionary используется только как финальный резерв для перевода, если английская запись не содержит русского варианта.
- Парсер выбирает значение структурно: учитывает часть речи, первую этимологию, глагольные формы, смысловые блоки перевода и отдельные страницы `/translations`. Если точная страница не дала результат, используется поиск MediaWiki по близким заголовкам.
- Новые проблемы поиска исправляются общими правилами разбора Wiktionary, а не исключениями для конкретных слов. Если перевод не найден, приложение показывает сообщение под полем слова, а пользователь может заполнить данные вручную.
- В «Примеры» добавляются примеры употребления и, для неправильных глаголов, формы прошедшего времени и причастия. Синонимы сохраняются отдельным полем, а другие значения подгружаются из Wiktionary и кэшируются на сессию.
- Парсер вынесен в [`wiktionary.js`](wiktionary.js); локальные тесты запускаются командой `deno test tests/wiktionary_test.js`.
- В таблице `public.cards` используются `id`, `word`, `translation`, `examples`, `synonyms`, `phonetic`, `created_at`. Для уже созданной базы выполните `alter table` из [`supabase/schema.sql`](supabase/schema.sql).
